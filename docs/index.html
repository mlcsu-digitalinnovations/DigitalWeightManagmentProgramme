Status

Manages status updates for referrals. Endpoints allow providers to update and retrieve the status for a referral identified by provider UBRN. The status of a referral can be set to one of the following:

  • Accepted: The referral has been successfully posted to the provider's API.
  • Completed: The service user has completed the provider's programme.
  • Contacted: The service user has been contacted by the provider.
  • Declined: The service user has decliened to start the provider's programme.
  • Rejected: The provider has rejected the user after inital contact.
  • Started: The service user has started the provider's programme.
  • Terminated: The service user has started the programme but has not completed it.

There are rules governing status progression:

  • Accepted can only be updated to Contacted, Rejected or Started.
  • Contacted can only be updated to Declined or Started.
  • Started can only be updated to Terminated or Completed.
  • Completed, Declined, Rejected and Terminated are all end states and a referral's status cannot be updated once these statuses have been reached because workflows are triggered upon their receipt.

A 409 conflict status code will be returned with the details of the problem should these rules be breached.
